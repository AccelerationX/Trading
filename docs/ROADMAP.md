# Roadmap

## Phase 0: Grounding

- 完成项目结构初始化
- 写清楚系统目标、边界和分层
- 明确“人机协同、不自动交易”的基本原则

## Phase 1: Information Source Design

- 定义信息源大类
- 定义每类信息在系统中的作用
- 划分必须收集、后续可加、暂不处理
- 明确来源可信度和时点要求
- 为高优先级信息源预留目录与接入口

## Phase 2: Data Contracts

- 定义市场数据、事件数据、新闻数据、人工记录的统一字段
- 定义快照和归档规则
- 定义每日运行输入输出格式

## Phase 3: Candidate + Context Fusion

- 接入历史量化候选池
- 建立市场环境判断模块
- 建立消息/公告事件结构化模块
- 生成第一版综合观察池

## Phase 4: Decision Drafting

- 生成每日建议卡片
- 输出买卖候选、仓位建议、失效条件和风险提示
- 建立人工确认后的交易记录回流流程

## Phase 5: Memory and Review

- 记录真实执行与未执行原因
- 建立相似案例检索
- 输出复盘和经验总结

## Phase 6: Iteration

- 检查建议质量
- 修正噪音源和低价值模块
- 优化提示词、规则和账户约束映射
