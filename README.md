# TradingSystem

面向 A 股个人交易者的人机协同分析系统。

这个项目不是自动交易引擎，也不是单纯的量化回测仓库。它的目标是把以下几类能力整合到同一个决策框架中：

- 历史量化研究与候选生成
- 实时消息、公告、政策、题材和市场语境理解
- LLM 辅助的事件解释、主线识别、风险提示和交易建议
- 交易者个人经验、复盘记录和决策记忆沉淀

## 设计原则

- 人在回路中：系统生成建议，最终交易由人工执行。
- 证据链优先：每条建议都必须能说明依据、条件和失效点。
- 真实约束优先：围绕实际资金、持仓上限、换手能力和执行精力设计。
- 分层清晰：研究、数据、LLM 分析、决策、复盘、输出彼此分离。
- 避免旧问题：不重演 `Trading` 的未来函数污染，也不重演 `TradingMain` 的理想化执行假设。

## 顶层结构

- `docs/`：总设计文档、模块说明、路线图
- `docs/INFORMATION_SOURCE_DESIGN.md`：信息源体系设计
- `docs/SOURCE_CATALOG.md`：来源家族目录
- `docs/DATA_CONTRACTS.md`：统一数据契约
- `configs/`：配置说明与后续配置文件
- `src/trading_system/`：系统主代码
- `data/`：原始数据、处理中间层、快照与参考数据
- `prompts/`：LLM 系统提示词和任务提示词
- `research/`：新系统专用研究、假设、实验与复盘
- `workspace/`：每日工作区、观察池、交易日志
- `workspace/templates/`：人工输入模板
- `outputs/`：系统产出，包括日报、建议、分析结果
- `scripts/`：命令行脚本与批处理入口
- `tests/`：测试
